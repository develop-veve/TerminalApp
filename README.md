# 生産管理　実績収集アプリ

---

## 実装目的・目標
　　
　１．アプリ作ったり、バーコード使ったり、ハンディターミナル使ったり
　　　そういう固定概念でしか作れない一番大切で速度が求められる現場生産実績について
　　　まとめ生産でも、一つに時間がかかる生産でも、どの単位でどの部品製品材料でも
　　　現場から全て発信するプログラム構成を検討する。
---

## 環境
- 言語：C#

- 画面: Microsoft.NET.Sdk.BlazorWebAssembly (Client),
             WebServer (ASP.NET Core WEB API)、電卓共通（クラスライブラリ)
- IDE：VisualStudio2022 （バージョンは問いません）

---

## 動作確認方法
- サーバーとクライアントを同時にデバッグ実行してください。
- 想定として、実績取集のWEBアプリを想定して作成。タッチパネルで操作。
- 製品名やその製品の状態を選択して、数量を登録、数量は電卓入力でイコールでセットされます。
  ※途中、極力操作なしで登録を検討中（１個１０秒で作成できるものなどの登録対応を検討中）。

---

## 今後実装予定の機能
- 数量入力のテキストボックスの桁数制限
- 各項目のコンボボックス以外の文字、数値その他入力制限
- チェック実行後のチェック対象項目へのフォーカス移動
- エンターキーで自項目へ進む
- テンキーが電卓に対応して動く（テンキーのエンターで数量反映）
- 保存後、ボタンを `Enable=false` にできていない。（不具合）
- 初期起動時に電卓画面をデフォルトで起動
- 製品の生産完了時（プラス）に、必要部品および構成品の在庫払い出し
- 製品の生産完了時（マイナス）に、必要部品および構成品の在庫戻し
